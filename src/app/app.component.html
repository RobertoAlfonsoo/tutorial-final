Guía para Principiantes e Intermedios

1. ¿Qué es Angular?

Angular es un framework de código abierto para desarrollar aplicaciones web.

Utiliza TypeScript, un superconjunto de JavaScript, para construir aplicaciones ricas en el lado del cliente.

Angular se destaca por su capacidad para crear Single Page Applications (SPA), donde la mayor parte de la interacción ocurre en una sola página, proporcionando una experiencia de usuario fluida y dinámica.

Beneficios de Angular para Desarrolladores Principiantes

  • Estructura clara y mantenible: Angular ofrece una estructura bien definida, facilitando a los nuevos desarrolladores seguir buenas prácticas desde el principio.
  • Reactividad y rendimiento mejorado: Gracias a su arquitectura basada en componentes, Angular permite crear interfaces de usuario altamente reactivas y eficientes.
  • Amplia comunidad y recursos: Al ser un producto de Google, Angular cuenta con una gran comunidad de desarrolladores y una amplia gama de recursos de aprendizaje.

¿Qué aplicaciones tiene Angular?

  • Aplicaciones web SPA: Angular permite crear aplicaciones donde la página no se recarga, mejorando la experiencia de usuario (ej. redes sociales).
  • Aplicaciones progresivas (PWA): Aplicaciones web que funcionan como nativas, con características como trabajar sin conexión (ej. Google Maps).
  • Dashboards interactivos: Herramientas visuales para mostrar datos en tiempo real y permitir la interacción (ej. paneles de ventas).
  • Aplicaciones móviles (con Ionic): Desarrolla aplicaciones móviles híbridas para Android y iOS con un solo código base usando Angular e Ionic.

¿Sobre qué lenguajes se apoya Angular?

  • TypeScript: Lenguaje principal de Angular, con tipado estático, lo que mejora la escalabilidad y detecta errores temprano.
  • HTML: Estructura la interfaz de usuario en los componentes, permitiendo una vista dinámica con directivas y enlaces de datos.
  • CSS/SCSS: Usado para el diseño visual de los componentes, con SCSS ofreciendo características avanzadas como variables y anidación.

2. Instalación

¿Que requisitos necesitaremos?

El framework Angular requiere la instalación previa de Node.js, npm y Angular CLI para gestionar dependencias y ejecutar comandos esenciales durante el desarrollo.

Pasos para realizar la instalación

El primer paso para trabajar con Angular es configurar tu entorno de desarrollo. Esto implica instalar Node.js, un entorno de ejecución para JavaScript que permite ejecutar scripts del lado del servidor.

Junto con Node.js, necesitarás npm (Node Package Manager), que es el gestor de paquetes de Node.js. npm facilita la gestión de las bibliotecas y herramientas necesarias para tus proyectos, incluyendo Angular.

La instalación de Node.js y npm es sencilla y puedes encontrar guías detalladas online para diferentes sistemas operativos.

¿Rutas para tener en cuenta?

El archivo src/app contiene el código principal. En ella se ubicarán los distintos componentes, servicios y páginas.

¿Variables de entorno?

En Angular 19, no es necesario usar variables de entorno de forma predeterminada. Puedes definir configuraciones directamente en servicios o componentes y acceder a ellas según sea necesario.

¿Necesita base de datos?

Angular no incluye una base de datos por defecto, pero puede integrarse con bases de datos externas usando APIs RESTful o GraphQL.

5. Explicación del Funcionamiento del Framework

¿Actúa sobre páginas, ficheros, secciones…?

Angular actúa sobre componentes, que contienen la lógica (TS), vista (HTML) y estilos (CSS). Las páginas se gestionan con el enrutamiento de Angular, donde cada componente puede representar una "sección" o vista de la aplicación.

¿Cómo se editan/modifican los ficheros que actúan en el framework?

Los ficheros de componentes (HTML, TS, CSS) se editan directamente. Los módulos (.module.ts) agrupan componentes y servicios, mientras que los servicios (.service.ts) se modifican para gestionar datos o lógica compartida.

¿Cómo se referencian los ficheros creados desde otras aplicaciones (si es el caso)?

Los componentes y servicios se importan en otros archivos usando import. Los módulos se importan en el app.component.ts del módulo principal para hacer que sus componentes estén disponibles en la aplicación.

6. Conclusiones

Este tutorial ha proporcionado una introducción fundamental sobre cómo trabajar con Angular, desde su instalación y configuración hasta la creación de una aplicación básica. Angular se presenta como un framework robusto y flexible, ideal para el desarrollo de aplicaciones web dinámicas y de alto rendimiento, gracias a su arquitectura basada en componentes, enrutamiento eficiente y capacidad para gestionar datos a través de servicios.

A lo largo del tutorial, hemos cubierto las bases del framework, cómo crear una aplicación, cómo estructurarla usando componentes y módulos, y cómo modificar y gestionar las vistas y lógica de la aplicación. También hemos explorado cómo Angular facilita la reutilización de código y la separación de responsabilidades, lo que simplifica el desarrollo de aplicaciones escalables.

Con los conocimientos adquiridos, puedes comenzar a desarrollar aplicaciones más complejas, implementar características avanzadas y optimizar el rendimiento de tus proyectos. Angular ofrece un ecosistema completo para el desarrollo web, y este tutorial te proporciona las herramientas necesarias para empezar a aprovechar todo su potencial.
